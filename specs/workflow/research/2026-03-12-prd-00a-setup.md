# Research: PRD-00a — Setup

**Date**: 2026-03-12
**Git Commit**: 88597f7
**Branch**: dev
**Repository**: gestao-projetos
**Research Mode**: Specs-only

## Research Question

Investigar o PRD-00a (Setup) no contexto do projeto: o que ele define, quais decisões de stack e versões estão pinadas, qual o estado atual do repositório, e o que precisa ser criado.

## Summary

O repositório está em fase pré-código. Contém apenas specs de foundation (6 arquivos), PRDs, e workflow prompts. Não existe `src/`, `package.json`, nem qualquer código de aplicação. O PRD-00a é o primeiro PRD da sequência e tem como objetivo inicializar o projeto Next.js com a stack base — ao final, `pnpm dev` e `pnpm build` devem funcionar sem erros.

## Detailed Findings

### 1. Estado atual do repositório

O repo contém exclusivamente documentação:

```
.gitignore
CLAUDE.md
specs/foundation/         — 7 arquivos (00 a 06)
specs/prds/               — roadmap, template, prd-00a-setup
specs/workflow/            — prompts (3) e research (.gitkeep)
```

**Nenhum arquivo de código existe:** sem `package.json`, `tsconfig.json`, `next.config.ts`, `src/`, nem dependências instaladas. O PRD-00a é literalmente o ponto zero de implementação.

### 2. Escopo do PRD-00a

**Objetivo:** Inicializar o projeto Next.js na raiz do repositório existente (que já tem `specs/` e `CLAUDE.md`).

**Inclui:**
- Inicialização Next.js 16 com TypeScript strict e App Router
- Tailwind CSS 4.x
- shadcn/ui (init + `cn()` helper)
- Estrutura de pastas com camadas vazias (`.gitkeep`)
- Layout raiz (`app/layout.tsx`) com metadata e fonte Inter
- Página placeholder em `/`
- Path aliases `@/`

**Não inclui:**
- Prisma, banco de dados, Supabase Auth (→ PRD-00b)
- `proxy.ts`, `requireAuth()` (→ PRD-00b)
- Layouts `(internal)` e `(portal)` (→ PRD-00b)
- Deploy, `vercel.json` (→ PRD posterior)
- Componentes de domínio, formulários, schemas Zod (→ PRDs seguintes)

### 3. Comando de inicialização

Definido na spec (`prd-00a-setup.md:66`):

```bash
pnpm create next-app . --typescript --tailwind --app --src-dir --import-alias "@/*" --use-pnpm
```

O flag `.` garante criação na raiz do repo (não cria subpasta).

### 6. Configurações específicas

**shadcn/ui (`prd-00a-setup.md:79-85`):**
- `pnpm dlx shadcn@latest init`
- Style: New York
- Base color: Neutral (ou Zinc)
- CSS variables: sim
- Gera `lib/utils.ts` com `cn()` e `components.json`

**Tailwind v4 (`prd-00a-setup.md:75-76`):**
- Diretiva `@import "tailwindcss"` no `globals.css`
- **Não** usar `@tailwind base/components/utilities` (sintaxe v3)

**Layout raiz (`prd-00a-setup.md:93-97`):**
- `<html lang="pt-BR">`
- Fonte: Inter via `next/font/google`
- Classes: fonte + `antialiased` no `<body>`
- Metadata: `title` (default) e `description`

### 7. Critérios de aceitação

Todos do `prd-00a-setup.md:113-121`:

1. `pnpm dev` inicia sem erros
2. `pnpm build` completa com zero erros TypeScript
3. `localhost:3000` exibe a página placeholder
4. `src/` segue a estrutura de camadas completa
5. `@/` resolve imports corretamente
6. `components.json` do shadcn/ui existe e configurado
7. Tailwind CSS funciona (classes utilitárias aplicam estilos)
8. TypeScript em modo strict

### 8. Relação com a arquitetura geral

O PRD-00a implementa apenas o esqueleto da estrutura definida em `specs/foundation/03_arquitetura.md`. A arquitetura completa prevê:

- **Layered architecture** com 7 camadas: app → components → actions → services → queries → lib → types
- **Fluxo canônico** de leitura (page → queries → db → components) e mutação (component → action → service → db)
- **Dois contextos**: `(internal)` autenticado e `(portal)` público — ambos criados no PRD-00b

O PRD-00a cria as pastas vazias para todas as camadas, mas só implementa `app/` (layout + page placeholder) e `lib/` (utils.ts).

## Open Questions

1. **Versão exata do Next.js 16.1.6:** O `create next-app` instalará a latest do Next.js 16. Se a versão disponível for diferente de 16.1.6 (a pinada nas specs), pode ser necessário ajustar o `package.json` após a inicialização.
2. **Compatibilidade shadcn + Tailwind v4:** O PRD assume que `shadcn@latest` já funciona com Tailwind v4. Se houver conflito no `init`, pode exigir ajustes manuais no `globals.css`.
3. **`.gitignore` existente:** O repo já tem `.gitignore`. O `create next-app` pode tentar sobrescrever — precisa preservar o conteúdo existente ou fazer merge.
